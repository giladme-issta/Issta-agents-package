Issta · Agent & Skill Workspace

Command Deck

A field map of how Aluf⭐ commands seven specialist agents, nine skills, and a file-based memory system across the Issta codebase — for any developer picking up this workspace cold.

What this actually is. Nothing here executes on its own — there is no runtime, server, or scheduler. This workspace is a set of GitHub Copilot custom chat agent definitions (*.agent.md frontmatter files), skill references (SKILL.md), and a hand-written file-based memory system, all wired together by convention through copilot-instructions.md. Every arrow on this page is an instruction one markdown file gives an LLM to read another markdown file, or to invoke another agent by name — enforced entirely by the model choosing to follow the written rules, not by code.

02

The Aluf⭐ Workflow

Every task run through Aluf follows this exact seven-step sequence, defined in agents/Aluf.agent.md. Skipping a step is treated as a defect, not an optimization.

01

Analyze & Plan

Identify domain, files, intent. Ask one focused question if ambiguous — never guess.

02

Memory Retrieve

Invoke Memory-Agent (RETRIEVE) with domain + task. Fold insights into the specialist prompt.

03

Delegate

Invoke specialist(s) in dependency order, each instructed to load issta-stack + its domain skills.

04

Code Review

Code-Reviewer checks every change. Disputes go through one arbitration round, then the ruling is final.

05

Rate Agents

rating-skill scores every invoked agent 1–5 against the task outcome.

06

Memory Store

Memory-Agent (STORE) writes the entry, appends the index row, and updates the ledger.

07

Final Report

A structured report — agents invoked, conflicts resolved, memory status — always closes the task.

05

Memory System

A plain-markdown, append-only knowledge base — no database, no service. Managed exclusively by Memory-Agent per skills/memory-structure/SKILL.md.

Lifecycle
  1. RETRIEVE — before a specialist starts, Aluf calls Memory-Agent with domain + task description. Read-only; never writes files.
  2. Work happens — retrieved insights are folded into the specialist's prompt so past mistakes aren't repeated.
  3. STORE — after the task, Memory-Agent writes an entry, appends one row to index.md, and appends reusable bullets to the domain file.
  4. Ledger — Aluf appends one row per invoked agent to agent-performance.md with its 1–5 score.
Score scale (rating-skill)
5Flawless — no review issues, no arbitration
4Good — minor issues fixed without conflict
3Acceptable — significant corrections or one arbitration round
2Poor — repeated failures, heavy intervention needed
1Failed — incorrect/harmful output, regression caused
File formats

index.md row — one per stored task, append-only:

| YYYY-MM-DD | entry-id | domain | [Agents] | task summary |

entries/YYYY-MM-DD_<slug>.md — frontmatter (id, date, agents, domain, outcome) followed by Task Summary, Agents Involved, Key Insights & Tricks, Outcome, Related Files.

domains/<domain>.md — living document, bullets only appended, never rewritten.

agent-performance.md — one row per agent per task: Date | Task | Agent | Score | Outcome | Notes. Corrections are new rows tagged [CORRECTION], never edits.

08

A Task, End to End

Worked example: “Add a new hotel filter that also needs GA4 tracking.” Domain = hotels + analytics, client-side.

  1. Developer messages Aluf⭐ directly — never a specialist first, per copilot-instructions.md §1.
  2. Aluf reads the request: domain hotels, Angular client, touches GA4 → routes to WebAgent-Expert-Hotel-Client.
  3. Aluf calls Memory-Agent in RETRIEVE mode: domain hotels, task “new filter + GA4”. Gets back any prior gotchas (e.g. index-based tracking bugs).
  4. Aluf delegates to WebAgent-Expert-Hotel-Client with the retrieved insights, instructing it to load issta-stack + angular-patterns + gtm-ga4-tracking from skill-route first.
  5. The specialist adds the FilterConfig entry, wires it into HotelFilterService, and pushes the GA4 event per the { ecommerce: null }-then-push convention.
  6. Aluf calls Code-Reviewer, which loads owasp-security + issta-stack and checks the diff — say it flags a 0-based GA4 index.
  7. The specialist fixes it directly (no valid challenge here — it's a documented convention) and Code-Reviewer re-reviews only the changed lines.
  8. Aluf applies rating-skill: WebAgent-Expert-Hotel-Client scores 4/5 (“minor GA4 index bug, fixed cleanly”), Code-Reviewer 5/5.
  9. Aluf calls Memory-Agent in STORE mode with the summary, insight (“GA4 index must be 1-based”), and outcome — new entry, index row, domain bullet, and two ledger rows all written.
  10. Aluf returns the structured Task Report to the developer: agents invoked, outcome, memory status — done.
